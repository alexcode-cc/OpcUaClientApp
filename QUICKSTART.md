# 快速開始指南

## 5 分鐘快速上手

### 步驟 1: 編譯並執行

```bash
cd OpcUaClientApp
dotnet restore
dotnet build
dotnet run
```

### 步驟 2: 連線到 OPC UA Server

1. 確認您的 OPC UA Server 正在執行（例如：OpcUaSqlServer）
2. 在應用程式中輸入端點位址：`opc.tcp://localhost:4840/OpcUaSqlServer`
3. 點選「連線」按鈕
4. 等待狀態燈變成綠色 🟢

### 步驟 3: 瀏覽節點

1. 連線成功後，節點樹會自動載入
2. 展開「Objects」節點
3. 瀏覽 SqlData 資料夾
4. 查看 Products 和 Orders 資料表節點

### 步驟 4: 讀取資料

1. 在節點樹中選擇一個變數節點（例如：Products_ProductName）
2. 點選「讀取」按鈕
3. 查看當前值和時間戳記

### 步驟 5: 寫入資料

1. 選擇一個可寫入的變數節點
2. 在「新值」欄位輸入要寫入的值
3. 點選「寫入」按鈕
4. 確認寫入成功

### 步驟 6: 監控資料變化

1. 選擇要監控的節點
2. 設定更新間隔（例如：1000 毫秒）
3. 點選「訂閱節點」按鈕
4. 觀察監控列表中的即時更新

## 常見使用場景

### 場景 1: 測試資料庫資料讀取

```
1. 連線到 OpcUaSqlServer
2. 瀏覽到 SqlData → Products → ProductName
3. 點選「讀取」查看資料庫中的產品名稱
```

### 場景 2: 更新資料庫資料

```
1. 選擇 Products → ProductName 節點
2. 輸入新的產品名稱
3. 點選「寫入」
4. 資料會寫入到資料庫
```

### 場景 3: 即時監控資料變化

```
1. 選擇要監控的節點
2. 設定更新間隔為 1000ms
3. 點選「訂閱節點」
4. 當資料庫資料改變時，會即時顯示
```

## 狀態燈號說明

- 🔴 **紅色**: 未連線 - 請檢查伺服器是否執行
- 🟡 **黃色**: 連線中 - 正在建立連線
- 🟢 **綠色**: 已連線 - 可以正常操作

## 快速鍵

- `F5`: 重新整理節點樹（需自行實作）
- `Ctrl+R`: 讀取選中的節點（需自行實作）
- `Ctrl+W`: 寫入選中的節點（需自行實作）

## 疑難排解速查

| 問題 | 解決方式 |
|------|---------|
| 無法連線 | 1. 檢查伺服器是否啟動<br>2. 確認端點位址正確<br>3. 檢查防火牆設定 |
| 節點樹為空 | 點選「瀏覽節點」重新載入 |
| 無法寫入 | 1. 確認節點支援寫入<br>2. 檢查資料型別是否正確 |
| 訂閱無更新 | 1. 確認更新間隔 ≥ 1000ms<br>2. 檢查資料是否真的在變化 |
